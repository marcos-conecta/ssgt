## Clean Architecture  
A ideia de criar o projeto usando o Clean Architecture foi para separar as responsabilidades e criar uma estrutura que seja fácil de entender, manter e escalar. Ela é baseada em princípios como a inversão de dependências, a separação de preocupações e a independência de frameworks.

O projeto é dividido em camadas, onde cada camada tem uma responsabilidade específica. 
- A camada de domínio contém as entidades e regras de negócio;
- A camada de aplicação contém os casos de uso;
- A camada de infraestrutura contém as implementações específicas, como repositórios e controladores.
